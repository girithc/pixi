# AGENTS.md

## File constraints

- Keep every file under 200 lines of code.
- Split files that exceed the limit into smaller modules.

## Structure

- Use a modular file structure.
- Build reusable components and share them across the codebase.
- Prefer composition and small, single-purpose units over large monolithic files.

## Design principles

- Follow Apple design principles: clarity, deference, and depth.
- Apply the Liquid Glass design framework in all UI work.
- Prioritize material, translucency, and layering that reflects content beneath.
- Keep visuals consistent, fluid, and responsive to user interaction.

## Code economy

- Do not write unnecessary code.
- Use as few lines as possible while keeping the code correct and readable.
- No dead code, no unused helpers, no redundant abstractions.
- Prefer existing utilities over reimplementing them.
